# DMR Documentation

## Development Environment

Use Nix (recommended): `nix develop` → `npm install`

Requires Node >= 22.12 (package.json specifies >=20.0 but README requires >=22.12 per Nix env).

**Security warnings**: `npm install` reports 29 vulnerabilities (28 moderate, 1 high). Run `npm audit fix` for auto-fixable issues, or `npm audit` to review manually. Some vulnerabilities may require choosing different dependencies.

## Commands

- `npm start` — dev server at localhost:3000, hot-reloads
- `npm run build` — builds static files; runs `prebuild` which generates `static/llms.txt` from docs
- `docusaurus` CLI for custom commands (deploy, clear, serve, write-translations, write-heading-ids)

## Architecture & Constraints

- Docusaurus 3.10.1 with classic preset; docs in `docs/`, sidebar defined in `sidebars.js` with autogenerated sidebar
- Spanish translations mirror structure under `i18n/es/docusaurus-plugin-content-docs/current/`
- Base URL: `process.env.BASE_URL ?? '/DMR/'` (set explicitly for production deploys)
- Mermaid diagrams included as text fences in `llms.txt` generation via `scripts/generate-llms-txt.mjs` (prebuild step)
- Frontmatter YAML stripped during llms.txt generation; body markdown kept verbatim

## Deployment

Automatic on push to master via `.github/workflows/deploy.yml`

Manual: `BASE_URL=/dmr_doc/ npm run build` then push `build/` to `gh-pages` branch

## Structure

- `docs/` — English docs source
- `i18n/es/docusaurus-plugin-content-docs/current/` — Spanish translations
- `src/` — homepage and custom components (components/, css/, pages/)
- `static/` — favicon, images, generated `llms.txt`, `llms-full.txt`
- `scripts/` — `generate-llms-txt.mjs` (run via prebuild)
